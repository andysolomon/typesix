“Typesix” is a vertical grid system composed of three great resources. The first component of Typesix is the Robert Bringhurst Typographic scale as noted in his legendary book, “The Elements of Typographic Style”. The second component is a six pixel vertical grid system developed by Teehan Lax back in 2011. Last but not least, Typesix uses Pearsonifiedâ€™s Golden Ratio to get the correct amount of characters per line (CPL) & appropriate line height for a readable paragraph.

Most designers set their type arbitrarily, either by pulling values out of the sky or by adhering to a baseline grid. The former case isn’t worth discussing here, but the latter requires a closer look.

Avoid using em tags when writing markup. The em tag adds an extra pixel to the line-height for reasons I'm not sure of. Try using an italic font-style, or use the true italic font-face of the desired typeface.

Numbers Added to the
Bringurst Typographic Scale

30, 42, 96, 120, 156

Why 30? 30 is the first number divisible by six immediately after 24. According to the iOS Design Guidelines 32px, is the optimal size for reading text on a mobile device. Consider this as a font-size for mobile experiences. Personaly I love using 30px for H2’s, and subheadings.

Why 42? 42 is chosen strictly has a line-height( a common theme when using Typesix ). Typesix sets its type based on two opionated ideals. The first is 960px is the optimal width for viewing and reading on the web desktop. The second strong opinion is 66 characters is the ideal line-width for reading text. Using the golden ration calculator, I've found the ideal line-height for a 30px font is actually 43px tall. Since 43 is not divisible by 6 we round down to the nearest number divisible by six. which is our lucky number 42

Why 96? 96 similiar to 42 is introduced because it's the ideal font size for the largest number on the Bringust Scale 72. Again using the same golden ratio rules as applied to 42. 96px is the optimial is the exact optimal line-height for 72px font. We strive to achive a 66 character line width within a 960px alloted space.

Why 120? 120 exceeds the Bringhust Scale and honestly is a number added strictly based upon the ideals of Typesix. 120px is largest type that I would typcally set on any website, and the number 120 is a number than can be be divided into many pieces.
Ultimately this means that 120 can be cut into ½, ⅓, ¼, ⅕, ⅙, ⅛. By the way, all of the numbers added to Bringhust Scale for Typesix can easily be divided by 4, which gives us the ability to add padding and margin based on fractions of the font-size. To learn more about padding and margin as related to Typesix and typography in general please take a look at MovePad.
